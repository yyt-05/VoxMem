# 声纹过滤方案

版本：v0.2
日期：2026-05-24

## 1. 问题

用户在录音说话时，旁边人同时说话，ASR 会把所有人的话都转成文字混在一起。

目标：只转写用户的声音，过滤其他人的声音。

## 2. 现状

```
浏览器 PCM → WebSocket → Go 后端 → WebSocket → 阿里云 Paraformer realtime → 文字
```

每 100ms 一帧 PCM 通过 Go 后端中转，无任何说话人过滤。

## 3. 已实现：RMS 能量过滤（第一层）

### 3.1 原理

你离麦克风最近 → 音量最高。远处的人音量低。

```
浏览器录 PCM → 计算 RMS → 自适应用户音量基线 → < 基线 25% → 丢弃
                                                → ≥ 基线 → 发 ASR
```

### 3.2 学习机制

- **快速上升**：RMS > 基线 × 1.2，直接更新基线（捕捉用户开始说话）
- **缓慢衰减**：否则基线每次 × 0.9998（防止误把静音当基线）
- **学习保护**：前 30 帧不执行过滤
- **激活条件**：基线 > 0.0005（确保不是纯静音）

### 3.3 代码位置

`web/src/App.tsx` — `startAudioProcessing` 函数内的 `onaudioprocess` 回调。

## 4. 第二层：speaker_id 说话人过滤（基础设施已就绪，等待阿里云支持）

### 4.1 设计

ASR 返回的每句文本都带 `speaker_id`，前端按说话人分组，用户可选择保留哪个说话人：

```
ASR → sentence + speaker_id → 前端分组 → 多人时显示选择器 → 选自己的 speaker → 只保留该 speaker 文本
```

### 4.2 已完成的代码

| 层 | 改动 |
|------|------|
| paraformer.go | `Sentence` 结构加 `SpeakerID`, `ExtractSentence` 返回 speaker_id |
| main.go | `clientEvent` 加 `SpeakerID`, 转发到前端 WebSocket |
| App.tsx | `speakerSegments` 按 speaker 分组, `hasMultipleSpeakers` 检测, radio 选择器 UI |

### 4.3 当前无法生效的原因

**paraformer-realtime-v2 实时流式 ASR 不支持说话人分离。**

- ASR 返回的 `speaker_id` 字段始终为 `null`
- 不是参数没开启——该模型就没有这个能力
- 阿里云说话人分离（cam++ 模型）仅在**离线文件转写**模式下可用

可验证：服务器日志中 ASR 返回的 JSON 原文：

```json
"sentence": {
    "speaker_id": null,
    ...
}
```

## 5. 方案对比（含调研结论）

| 方案 | 原理 | 可行性 | 已实现 |
|------|------|--------|--------|
| A. RMS 能量过滤 | 远处声音音量低，近处高 | ✅ 已实现 | ✅ |
| B. 阿里云声纹 + 实时 ASR | run-task 带 voiceprint_id | ❌ Paraformer 实时版不支持 | ❌ |
| C. speaker_id 后处理 | ASR 返回时带说话人标签 | ❌ 实时版 speaker_id = null | ⚠️ 代码已就绪 |
| D. 本地声纹模型 | 浏览器端跑 ONNX | ❌ 模型大、延迟高、精度差 | ❌ |

## 6. 可行的替代路径（未实现）

如果要做到真正的说话人区分，需要放弃实时流式，改用离线文件转写：

```
浏览器录完整音频 → POST /api/transcribe → Go → 阿里云文件转写 API (paraformer + cam++)
    → 返回结果带 speaker_id → 前端按说话人过滤
```

### 6.1 代价

- **延迟**：从毫秒级实时出字变成秒级文件转写（3-10 秒音频 + API 耗时 1-3 秒）
- **API 切换**：从 WebSocket 流式 paraformer-realtime-v2 切换到 HTTP 文件转写 paraformer-zh + cam++
- **后端改动**：新增文件转写接口，音频文件管理

### 6.2 FunASR 离线说话人分离模型组合

| 模型 | 用途 |
|------|------|
| paraformer-zh | 离线语音识别 |
| cam++ | 说话人分离/验证 |
| fsmn-vad | 语音活动检测 |

来源：阿里云开发者社区 / FunASR GitHub

### 6.3 是否值得做

- 如果产品定位是「实时出字输入法」→ 不适合，延迟太高
- 如果产品定位包含「会议记录/整理」→ 值得作为独立功能
